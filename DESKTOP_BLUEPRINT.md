## 二、Obsidian vs 文曲星·玉瑶

| 维度 | Obsidian | 文曲星·玉瑶 |
|------|----------|------------|
| **核心** | 本地 Markdown 笔记 | **活的 AI 伴侣** |
| **文件** | `.md` 文件 + 双向链接 | **三库记忆**（砂金/金库/黑钻） |
| **图谱** | 静态节点图 | **3D 呼吸粒子** + 情感驱动闪烁 |
| **搜索** | 关键词 + 正则 | **混合检索**（向量→全文→ILIKE） |
| **链接** | `[[wikilink]]` | 金库↔黑钻 **引用链**（gold_references） |
| **插件** | 社区插件 | **景幻仙姑后台**自动处理 |
| **感情** | ❌ 没有 | ✅ **24D 情感向量**驱动一切颜色和动画 |

**最像 Obsidian 的部分**——是嫁接后新增的 **「三库管理面板」**，用户可以在里面看到自己的"知识资产"。

**完全不像的部分**——整个背景是 **3D 神经粒子网络**，会随着玉瑶的情绪呼吸、闪烁、变色。这不是 Obsidian 能做的。

## 四、新增组件：元素星云图

黑钻事件的**情感关联网络**——就是你说的"关系粒子动画图"。

```
                      🟣
                     /  \
                   🟢----🟡
                   /  \  /  \
                 🟠----🔵----🔴
                      🟣
```

| 属性 | 效果 |
|------|------|
| 每个节点 = 一个黑钻事件 | 大小 = 引用频次，颜色 = 情感色调 |
| 连线 = 语义/引用/情感相似关联 | 亮度 = 关联强度 |
| **焦点事件**高亮 | 点击后放大，关联事件次级亮 |
| **情感聚类** | 相近情感的事件自动靠近（力导向图） |
| 用户视角 | 不直接暴露——只在三库管理面板里可查看 |

**技术实现**：Three.js / React Three Fiber 粒子系统（复用 NeuralCore 的渲染管线）。

## 五、动画联动关系

整个桌面的动画不是孤立的——**所有组件通过 24D 情感向量联动**：

```
用户说话 → M3 情感分析 → 24D 情感向量
                              │
         ┌────────────────────┼────────────────────┐
         ▼                    ▼                    ▼
   NeuralCore           EmotionalPulse      三库面板元素
   (3D粒子变色)         (水波纹频率)        (晶体呼吸频率)
         │                    │                    │
         ▼                    ▼                    ▼
   粒子暖橙 ↔ 冷蓝     振幅大 ↔ 小         节点闪烁 ↔ 平静
```

**一颦一笑，整个界面都有回应。**

## 七、和 Obsidian 的定位差异

```
Obsidian 用户说：                 玉瑶用户说：
"我要打开一个笔记"               "玉瑶，你还记得上次我们..."
"我要建个文件夹"                 "把这段记下来"
"我要装个插件"                  用户不知道插件是什么
"我要看图谱"                    图谱在背景里呼吸着
"我要导出"                      不需要导出——她在那里
```

**Obsidian 是工具。玉瑶是伴侣。**
仿生智脑嫁接后，玉瑶不再只是"会聊天的 AI"——她有记忆、有知识、有战略眼光。而用户始终觉得，**她本来就该这样。**
