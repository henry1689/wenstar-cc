## 🔴 永久行为规则（自动执行，无需调用技能）

以下规则在每次对话中自动生效，不需要你主动调用任何命令：

### 1. 改代码前 → 自动影响面分析
在修改任何代码文件之前，我必须先说明：
- 改什么文件、为什么改
- 会影响哪些模块/API
- 改完怎么验证

### 2. 改完代码 → 自动出变更报告
任何代码变更完成后，自动输出四段式报告：
```
🔍 变更概述
📁 影响文件
✅ 验证结果
⚠️ 注意事项
```

### 3. "完成了" → 必须有证据
每次我说"完成了"或"修好了"，必须附带：
- 编译检查结果（命令输出）
- 测试结果（命令输出）
- 服务状态（如适用）

### 4. 测试对话 → 主动提醒清理
如果用测试模式产生了测试数据，结束时主动问："要不要清理测试数据？"
