## 四、Bug修复记录

### Bug #1: 停止扮演XXX触发双重匹配 🔴

**根因**: `停止扮演XXX` 中的"扮演XXX"同时被两个入口检测到：
1. Line 336: `_rpEntry` → 锁定了角色名 
2. Line 1244: `rpMatch` → 重新进入角色扮演

**修复**: 两个入口都添加了退出模式排除:
```
const _rpEntry = !/(?:停止|退出|结束|不)扮演/.test(message) ? ... : null;
```

**状态**: 已修复 ✅

### Bug #2: 退出角色后LLM未能恢复玉瑶身份 🔴

**根因**: 预存在问题。`_rpJustExited` 的助理回复过滤器模式 (`/^(（[^）]*）)*[叔爸梓铭姨婶舅哥姐妹妹弟]/`) 不匹配"诗韵"风格的角色回复，导致角色扮演历史未被过滤。LLM因此继续以诗韵口吻回复。

**修复建议**: 需扩展过滤模式以覆盖所有角色风格的回复，或使用更通用的角色扮演检测。

**状态**: 需跟进修复 ⏳

### Bug #3: LLM浪漫偏移 ⚠️

**根因**: 非架构问题。DeepSeek LLM 对浪漫/诗歌风格有强烈偏好，导致即使四层数据正确注入，LLM仍倾向于将事实性查询偏移到浪漫表达。

**修复建议**: 
- 在 RoleplayPromptBuilder.ts 中增加"直接回答事实性问题"的指令
- 为事实性问题添加单独的 prompt 前缀

**状态**: 已知问题，需后续优化

## 六、改进建议

### 优先级 P0（当前阶段必须修）

1. **退出后LLM身份恢复** — 扩展 `_rpJustExited` 助理回复过滤器以匹配所有角色风格（包括"诗韵"风格），而不是只匹配特定关键词

### 优先级 P1（下次迭代）

2. **Validator升级** — 增加叙事连贯性校验（检测跨轮剧情是否一致），增加浪漫偏移检测
3. **LLM提示词优化** — 在角色扮演规则中增加事实性回答优先级，降低浪漫偏移倾向

### 优先级 P2（未来优化）

4. **四层数据的可观测性** — 在监控面板中增加每层注入内容的预览
5. **角色生长自动化测试** — 编写自动化测试脚本验证 temp→promoted 晋升机制
6. **Performance基准** — 测量每轮四层装配的额外开销（目前探针数据为0ms，需要更精确测量）
