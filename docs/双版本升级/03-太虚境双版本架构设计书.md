## 二、模块依赖关系

### 2.1 核心流水线依赖（严格单向）

```
M1 ──→ M3 ──→ M4 ──→ M5 ──→ PostProcess
 │              │              │
 │              │              ├──→ M6 (单向反馈)
 │              │              ├──→ M7 (异步触发)
 │              │              ├──→ M8 (单向写入)
 │              └──→ Vault (三库读写)
 │              └──→ FamilyGraph (读写)
 │              └──→ KnowledgeBase (读写)
 │
 └──→ LLMEntityExtractor (条件触发)
```

**依赖原则**:
- 上层模块不得引用下层模块（M1 不能引用 M3）
- 后处理模块不得反向影响主流水线输出
- 太虚图书馆不得引用任何主程序模块

### 2.2 版本差异依赖

| 模块 | 轻量版 | 旗舰版 |
|------|--------|--------|
| M1-M9 | ✅ 全量 | ✅ 全量 |
| 三库存储 | ✅ 全量 | ✅ 全量 |
| 人物档案 | ✅ 全量 | ✅ 全量 |
| 关系图谱 | ✅ 全量 | ✅ 全量 |
| 知识库 | ✅ 原生 | ✅ 原生 + 同步图书馆 |
| VAD 谱曲 | ✅ 支持 | ✅ 支持 |
| Bionic 记忆 | ✅ 支持 | ✅ 支持 |
| 外挂文件夹 | ❌ 不支持 | ✅ 自动摄入 |
| 太虚图书馆 | ❌ 不包含 | ✅ 内置运行 |
| MCP 服务 | ✅ 仅客户端 | ✅ 服务端+客户端 |
| 文档解析 | ❌ 不内置 | ✅ 全格式解析 |

## 三、数据流架构

### 3.1 对话处理流 (Synchronous)

```
步骤 1: 用户输入
    │
步骤 2: M1 DNA 编码
    ├── 规则路由 (L0)
    ├── 分支序列 (L1)
    ├── 叶子区指针 (L2)
    └── 实体基因 (L3)
    │
步骤 3: LLM 实体提取 (条件触发)
    └── 仅当 L3 提取到 0 个人名
    │
步骤 4: M3 24D 感知
    ├── 24 维情感分析
    ├── 钙化分计算
    └── 动作决策
    │
步骤 5: 记忆门控 (MemoryGate)
    ├── 模式分类 (casual/recall/knowledge)
    └── 构建守卫
    │
步骤 6: 检索管线 (Retrieval Pipeline)
    ├── 三库情感检索
    ├── VAD 谱曲引擎 (缓存检查)
    ├── 人物图谱关联
    ├── 黑钻 FTS5 检索
    ├── Bionic 记忆检索
    └── 知识库加权搜索
    │
步骤 7: M4 知识融合
    ├── 多源融合
    └── 上下文构建
    │
步骤 8: M5 表达生成
    ├── 认知组装
    ├── DeepSeek LLM 调用
    └── RoleGuard 校验 (含重试 + 降级)
    │
步骤 9: 后处理 (异步)
    ├── 对话持久化
    ├── 人物档案更新
    ├── 关系图谱同步
    ├── 主题追踪
    ├── M6 反馈
    ├── M8 年轮
    └── 高钙记忆晋升 (后台)
```

### 3.2 后台调度流 (Asynchronous)

```
MemoryAssessor 调度循环
    │
    ├── 每 30 分钟:
    │   └── 砂金→金库 晋升检查
    │       ├── SELECT FROM conversations WHERE calcium >= 1 AND is_promoted = 0
    │       ├── 对每条:
    │       │   ├── 生成 24D 情感向量
    │       │   ├── 写入 memories 表
    │       │   └── 标记 is_promoted = 1
    │       └── 记录晋升日志
    │
    ├── 每 2 小时:
    │   └── 金库→黑钻 晋升检查
    │       ├── SELECT FROM memories WHERE (calcium >= 4.5 OR recall_count >= 5) AND is_promoted = 0
    │       ├── 对每条:
    │       │   ├── 生成摘要
    │       │   ├── 写入 black_diamond 表
    │       │   ├── 写入 FTS5 索引
    │       │   └── 标记 is_promoted = 1
    │       └── 记录晋升日志
    │
    ├── 每 24 小时:
    │   ├── 钙化分衰减
    │   │   ├── UPDATE memories SET calcium = max(0, calcium - decay_rate)
    │   │   │   WHERE decay_type = 'neutral' ...
    │   │   └── UPDATE memories SET calcium = max(0, calcium - slow_decay)
    │   │       WHERE decay_type = 'emotional' ...
    │   │
    │   ├── 梦境归纳 (M7)
    │   │   └── 取出高钙记忆 → 生成认知块
    │   │
    │   └── 自检报告 (景幻仙姑)
    │       └── 三库统计 + 晋升汇总 + 异常告警
    │
    └── [旗舰版] 实时:
        └── 外挂文件夹变更监控
            └── 新增/修改文件 → 触发解析流水线
```

### 3.3 合并数据流（旗舰版，含太虚图书馆）

```
高价值对话 (calcium >= 3)
    │
    ├── 主程序: 正常写入三库
    │
    └── 太虚图书馆:
        ├── 通过 HTTP 3737 同步
        ├── 生成标准 MD 词条
        ├── 存入 Wiki 层
        └── 更新索引

主程序检索时:
    ├── 原生三库检索 (优先)
    └── 太虚图书馆检索 (补充)
        └── HTTP GET /api/v1/entries?q=... → 结果融合

外挂文件夹新增文档:
    ├── 图书馆解析 → 生成 MD
    ├── 生成 DNA 根码
    └── 主程序可通过 RAG 命中
```

## 四、并发与状态管理

### 4.1 组件状态策略

| 组件 | 状态存储 | 并发策略 | 说明 |
|------|----------|----------|------|
| VAD 健康 | 模块级变量 | 原子读写 | 首次失败永久标记 |
| WordHitCounter | Map 实例 | 无并发保护 | 每 1000 次自动清理 |
| 候选人 | ChatContext | 会话隔离 | 移出 globalThis |
| LLM 实体缓存 | Map 实例 | 3 分钟 TTL | 200 条上限 |
| 同轮向量缓存 | ChatContext | 单轮仅一次 | 避免重复计算 |
| TransitionManager | 实例级 | 熔断重置 | 30 分钟自动衰减 |

## 五、架构约束

### 5.1 模块物理隔离

```
主程序 (src/)        太虚图书馆 (lib/taixu-library/)
─────────────        ───────────────────────────────
src/m1/              lib/taixu-library/src/core/
src/m2/              lib/taixu-library/src/storage/
src/m3/              lib/taixu-library/
src/...
                     ❌ 不得 import 任何 src/ 模块
```

### 5.2 编译隔离

| 版本 | 编译命令 | 输出 |
|------|----------|------|
| 主程序 | `tsc -p tsconfig.json` | `dist/` |
| 太虚图书馆 | `tsc -p lib/taixu-library/tsconfig.json` | `lib/taixu-library/dist/` |
| 轻量版打包 | `scripts/package-lite.ts` | `dist/lite/` |
| 旗舰版打包 | `scripts/package-flagship.ts` | `dist/flagship/` |

### 5.3 数据契约

主程序和太虚图书馆通过以下数据契约互联：

1. **DNA 编码格式**: `DNA-YYYYMMDD-HHmm-NNNN-HX` （24 字符）
2. **钙化分公式**: 完全一致
3. **YAML 元标签字段**: title, type, calcium, entities, tags, summary, dna_root_id 等
4. **情感向量格式**: 24 维 float32 数组
5. **HTTP 接口协议**: JSON over REST

## 六、安全架构

### 6.1 数据安全

| 层面 | 措施 |
|------|------|
| API Key | 环境变量 + 可选 AES-256-GCM 加密存储 |
| 数据库 | SQLite 文件权限限制，不监听外部端口 |
| 第三方服务 | 仅通过 localhost 与 VAD(8100)/Bionic(7200) 通信 |
| 文件上传 | 文件类型校验，路径穿越防护 |
| 日志 | 脱敏处理（不记录 API Key 原文）|

### 6.2 输出安全 (RoleGuard)

| 角色 | 禁止输出 | 强制输出 |
|------|----------|----------|
| 秘书 | 亲密/性暗示词汇 | 职业化距离 |
| 回忆者 | 推测性表述 | 「我不太确定」/「不记得」 |
| 玉瑶 | 无（允许亲密） | 符合角色描述 |
| 导师 | 无 | 建设性意见 |
| 降级态 | 所有亲密词汇 | 秘书级别回复 |

```

[旗舰版] 太虚图书馆 ──MCP服务端──→ Obsidian 客户端
```

> **架构设计书是可执行蓝图。每一根箭头代表一条数据流，每一个方块代表一个独立的编译单元。**
> 按照这个架构，不会出现模块间循环依赖，不会出现版本间的功能泄漏。
