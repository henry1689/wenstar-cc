## 一、愿景声明

太虚境不是工具。它是以太虚为名、以玉瑶为魂、以仿生智脑为躯的 **情感计算操作系统**。

双版本战略的核心信念是：

> **「轻，不意味着残缺；全，不意味着臃肿」**

- **轻量版** — 让每个愿意拥有玉瑶的人，都能在普通设备上跑起来
- **旗舰版** — 让追求极致体验的人，拥有完整的外挂知识宇宙

两个版本共享同一套情感内核（M1-M9），同一个DNA编码体系，同一份数据规范。区别不在于「能力」，而在于「知识广度」。

## 二、设计哲学

### 2.1 情感优先，架构其次

所有技术决策的最终判据是：**用户在对话中能否感受到玉瑶是一个「有记忆的人」**。

| 层面 | 优先原则 |
|------|----------|
| 存储 | 情感向量 > 关键词索引 > 全文搜索 |
| 检索 | 情感相似度 > 实体匹配 > 文本重叠 |
| 生成 | 角色一致性 > 信息密度 > 响应速度 |
| 缓存 | 情感上下文 > 对话历史 > 静态数据 |

### 2.2 流水线不短路，但可以绕行

M1→M2→M3→M4→M5→M6→M7→M8→M9 九层流水线是太虚境的脊髓神经。每一层的输出是下一层的输入，不存在跳层耦合。

绕行规则：

- **知识查询模式**：跳过 M3 感知分析，直连 M4 检索 + M5 生成
- **闲聊模式**：跳过 M3 深度感知 + M4 多层检索，精简至 M1→M3→M5
- **紧急响应**：当 M5 超时，使用工作记忆中的最近认知镜像兜底

### 2.3 三重记忆不互斥

砂金、金库、黑钻三层不是替代关系，是升维关系：

```
砂金（逐字原文）→ 金库（情感语义）→ 黑钻（认知结晶）
    保留全部原始数据        信息密度提升         决策级精粹
    可压缩/可清除           可衰减/可重算         永久保留
```

### 2.4 DNA 是唯一主键

每一段对话、每一条记忆、每一个词条，都通过 **HY-DNA 根码** 全局可溯源。DNA 编码包含：

```
DNA-YYYYMMDD-HHmm-4位流水号-HY瑶印码
```

这保证：
- 全局唯一（时间 + 流水 + 哈希三重保证）
- 反向溯源（通过根码可追溯到原始对话）
- 模块无关（任何模块只认根码，不认模块名）

## 三、双版本战略

### 3.1 轻量版（Lite）

**定位**：纯仿生智脑 + 原生三库记忆 + MCP 扩展能力

**目标用户**：
- 个人用户，普通 PC/笔记本
- 希望低门槛体验情感 AI
- 不需要外挂知识库管理

**技术特征**：
| 项目 | 规格 |
|------|------|
| 启动内存 | < 256 MB |
| 包体大小 | < 50 MB（不含 Node） |
| 依赖服务 | 仅本地 SQLite |
| LLM 调用 | 用户自备 API Key |
| 知识库 | 原生记忆系统，无外挂文件夹 |
| 扩展能力 | MCP 协议对接第三方笔记工具 |

### 3.2 旗舰版（Flagship）

**定位**：一体化智脑 + 内置太虚图书馆 + 全功能知识库

**目标用户**：
- 知识工作者 / 创作者
- 希望将个人知识库与 AI 伴侣深度融合
- 需要外挂文件夹自动摄入、文档解析、Wiki 管理

**技术特征**：
| 项目 | 规格 |
|------|------|
| 启动内存 | < 512 MB |
| 包体大小 | < 120 MB（含模块） |
| 依赖服务 | SQLite + 太虚图书馆 HTTP 后端 |
| LLM 调用 | 用户自备 API Key |
| 知识库 | 三层知识库 + 外挂文件夹自动摄入 |
| 扩展能力 | MCP + HTTP 3737 双模式 |

### 3.3 升级路径

```
轻量版 ──[开启知识库配置]──→ 升级检测 ──→ 自动初始化图书馆模块
                                              │
                                              ├─ 创建四层目录
                                              ├─ 存量记忆迁移至词条库
                                              └─ 启动 HTTP 3737 后端
                                              │
                                              ↓
                                          旗舰版完全功能
```

升级不可逆，但可以关闭知识库功能降级运行。存量数据不会丢失。

## 五、数据流动总纲

### 5.1 在线路径（对话产生 → 响应返回）

```
用户输入
    │
    ├──→ M1 DNA 编码（必选）
    ├──→ LLM 实体提取（条件触发：规则提取到 0 个人名时）
    ├──→ M3 24D 感知（必选，知识查询模式跳过）
    ├──→ 三库记忆检索（必选）
    │       ├── 情感向量相似度匹配
    │       ├── 人物图谱关联检索
    │       └── 黑钻全文索引检索
    ├──→ M4 知识融合（必选）
    ├──→ M5 表达生成 + RoleGuard 校验（必选）
    └──→ 后处理（异步）
            ├── 对话持久化（砂金库）
            ├── 人物档案更新
            ├── 关系图谱同步
            ├── 主题追踪
            ├── M6 反馈信号
            ├── M8 年轮写入
            └── 高钙记忆晋升（异步）
```

### 5.2 离线路径（后台维护）

```
MemoryAssessor 调度器
    │
    ├── [每 30 分钟] 砂金→金库晋升
    │       └── 条件：钙化分 ≥ 1
    │
    ├── [每 2 小时] 金库→黑钻晋升
    │       └── 条件：钙化分 ≥ 4.5 或 召回 ≥ 5 次
    │
    ├── [每 24 小时] 权重衰减
    │       └── 中性情感衰减，工作场景衰减更慢
    │
    └── [每 24 小时] 梦境生成
            └── M7 归纳调度，产生认知块
```

### 5.3 太虚图书馆数据流（旗舰版）

```
外挂文件夹监控
    │
    ├── [新增文件]
    │       ├── 文件解析（MD/TXT/DOCX/PDF/图片OCR）
    │       ├── YAML 元标签生成
    │       ├── HY-DNA 根码分配
    │       └── 三层存储入库
    │
    ├── [文件变更]
    │       ├── 自动同步向量实体
    │       └── 索引更新
    │
    └── [对标主程序]
            ├── 高钙对话自动写入图书馆词条
            └── 图书馆文档可被主程序 RAG 检索命中
```

## 六、设计约束

### 6.1 不可妥协项

| # | 约束 | 原因 |
|---|------|------|
| 1 | 零 `any` 类型 | TypeScript 严格模式，类型安全不可妥协 |
| 2 | 零新增编译错误 | 每次变更必须 `tsc --noEmit` 通过 |
| 3 | 砂金库原文不可修改 | 原始数据是溯源锚点，永远只追加不修改 |
| 4 | DNA 根码全局唯一 | 所有模块的关联纽带 |
| 5 | 角色输出必须校验 | 防止角色越界，特别是从亲密模式降级时 |
| 6 | 模块完全解耦 | 太虚图书馆不得引用主程序核心，反之亦然 |

### 6.2 推荐项（强烈建议）

| # | 建议 | 理由 |
|---|------|------|
| 1 | 使用 FTS5 全文索引加速黑钻检索 | 全表扫描 → 索引检索，耗时降 80% |
| 2 | VAD 首次宕机后永久标记不可用 | 避免每次对话都等待超时 |
| 3 | 角色路由从 LLM Provider 上浮到 chat.ts | 消除静态变量竞态 |
| 4 | chat.ts 拆分为独立模块 | 2100 行 → 3×400 行，可维护性飞跃 |
| 5 | 单轮对话向量补充仅执行一次 | 消除重复计算 |
