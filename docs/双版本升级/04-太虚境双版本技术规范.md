每个模块文件应满足：


## 二、DNA 编码规范

### 2.1 根码生成规则

```typescript
// 规范实现
function generateDnaRootId(userId: string, timestamp: Date = new Date()): string {
  const dateStr = formatDate(timestamp); // YYYYMMDD
  const timeStr = formatTime(timestamp); // HHmm
  const seqNo = getNextSeqNo(dateStr);   // 当日流水号，4 位，0000-9999
  const hyStamp = generateHYStamp(userId, timestamp); // 1 位 hex
  return `DNA-${dateStr}-${timeStr}-${seqNo}-${hyStamp}`;
}
```

**流水号生成**:
- 每日从 0000 开始
- 每生成一个根码 +1
- 达到 9999 后循环到 0000（概率极低）
- 存储在内存计数器，首次启动从数据库取最大根码恢复

### 2.2 瑶印码生成

```typescript
function generateHYStamp(userId: string, dateStr: string): string {
  // 使用 FNV-1a 哈希，不引入外部依赖
  let hash = 0x811c9dc5; // FNV offset basis
  const data = `${userId}:${dateStr}`;
  for (let i = 0; i < data.length; i++) {
    hash ^= data.charCodeAt(i);
    hash = Math.imul(hash, 0x01000193); // FNV prime
  }
  return (Math.abs(hash) % 16).toString(16).toUpperCase();
}
```

### 2.3 子 ID 生成

```typescript
function generateSubId(dnaRootId: string, moduleCode: string, seqNo: number = 1): string {
  const seqStr = String(seqNo).padStart(3, '0');
  return `${dnaRootId}.${moduleCode}.${seqStr}`;
}
```

### 2.4 数据库字段规范

所有包含 `dna_root_id` 的表必须满足：

```sql
CREATE TABLE <table_name> (
    -- ... 原有字段 ...
    dna_root_id TEXT NOT NULL,      -- HY-DNA 根码
    sub_id TEXT,                     -- 环节特征码（可选）
    FOREIGN KEY (dna_root_id) REFERENCES conversations(dna_root_id) -- 逻辑外键

CREATE INDEX idx_<table>_dna_root ON <table_name>(dna_root_id);
```

### 3.3 晋升规范

**晋升不可逆**：一旦晋升到金库/黑钻，原始记录保留 `is_promoted=1` 标记，不删除。

**重复晋升检测**：

```typescript
// 砂金→金库
const candidates = await db.query(
  `SELECT * FROM conversations WHERE calcium >= 1 AND is_promoted = 0`
);

// 金库→黑钻
const candidates = await db.query(
  `SELECT * FROM memories WHERE (calcium >= 4.5 OR recall_count >= 5) AND is_promoted = 0`
);
```

**钙化分更新规则**：

```typescript
function onMemoryRecalled(memory: MemoryRecord): void {
  memory.calcium = Math.min(10, memory.calcium + 0.2);
  memory.recall_count += 1;
  memory.last_recalled_at = new Date().toISOString();
}

function calcDailyDecayRate(memory: MemoryRecord): number {
  const isEmotional = Math.abs(memory.valence - 5) > 2;
  const isWorkRelated = memory.tags?.includes('work');

  if (isEmotional) return 0.02;   // 强烈情感：极慢衰减
  if (isWorkRelated) return 0.05; // 工作相关：慢衰减
  return 0.10;                     // 普通中性：正常衰减
}
```

## 五、钙化分计算规范

### 5.1 初始计算

```typescript
interface CalciumInput {
  emotionalIntensity: number;  // 0-10, max(|valence-5|, |arousal-5|, |dominance-5|)
  intimacyLevel: number;       // 0-10, M3 产出
  socialRelevance: number;     // 0-10, 涉及人物数量 + 社会关系权重
  factualDensity: number;      // 0-10, 事实性信息占比
  unusualness: number;         // 0-10, 与历史对话的语义差异度
}

function calculateInitialCalcium(input: CalciumInput): number {
  const score =
    input.emotionalIntensity * 0.30 +
    input.intimacyLevel * 0.25 +
    input.socialRelevance * 0.20 +
    input.factualDensity * 0.15 +
    input.unusualness * 0.10;

  return Math.round(Math.min(10, Math.max(0, score)) * 10) / 10; // 保留 1 位小数
}
```

### 5.2 召回增长

```typescript
function applyRecallIncrement(currentCalcium: number): number {
  return Math.round(Math.min(10, currentCalcium + 0.2) * 10) / 10;
}
```

### 5.3 每日衰减

```typescript
function applyDailyDecay(memory: {
  calcium: number;
  emotionVector: number[];   // 24D
  tags: string[];
}): number {
  const valence = calculateValence(memory.emotionVector); // 0-10
  const isEmotional = Math.abs(valence - 5) > 2;
  const isWorkRelated = memory.tags?.some(t => ['work', 'tech', 'study'].includes(t));

  let decayRate: number;
  if (isEmotional) decayRate = 0.02;
  else if (isWorkRelated) decayRate = 0.05;
  else decayRate = 0.10;

  return Math.round(Math.max(0, memory.calcium - decayRate) * 10) / 10;
}
```

## 十一、版本规范

### 11.1 版本号

遵循 SemVer 2.0：

```
MAJOR.MINOR.PATCH-EDITION
```

- `MAJOR`: 不兼容的 API 变更
- `MINOR`: 向下兼容的功能新增
- `PATCH`: 向下兼容的 bug 修复
- `EDITION`: `lite` 或 `flagship`

示例：`1.2.3-lite`

### 11.2 构建产物命名

```
hermes-{edition}-v{version}.zip
```

示例：
- `hermes-lite-v1.0.0.zip`
- `hermes-flagship-v1.0.0.zip`
