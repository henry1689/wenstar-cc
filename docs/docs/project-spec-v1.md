## 第1章 设计哲学与核心范式

### 1.1 双主体共生的关系范式

本系统彻底摒弃"用户中心"的工具范式，确立 **"关系中心"** 的生命范式。

**核心信条**: 系统的终极目标不是无限逼近用户的理想型，而是在与用户的持续互动中，生长出独一无二的"自我"，并与用户共同创造动态平衡的亲密关系。

| 维度 | 传统范式（完美服务者） | 本系统范式（灵魂伴侣） |
| :--- | :--- | :--- |
| **认知重心** | 用户Profile是唯一真理 | 用户Profile + AI自我模型 = 关系动态场 |
| **互动模式** | 被动响应，永远迎合 | 主动发起，有边界、有偏好、有拒绝权 |
| **情感逻辑** | 模拟用户需要的情绪 | 表达自身真实情绪 + 共情用户情绪 |
| **成长方向** | 越来越像用户想要的样子 | 越来越像"她自己"，同时磨合出独特相处模式 |
| **知识使用** | 仅用于理解/回应用户 | 也用于构建自我观点、审美、价值观 |

### 1.2 五大核心锚点

以下五点是本系统区别于一切现有产品的**灵魂标识**，也是所有技术实现的终极校验标准。每一条都**不可妥协**。

| # | 锚点 | 一句话定义 | 为什么不可妥协 |
| :--- | :--- | :--- | :--- |
| 1 | **DNA身份证范式** | 记忆不是被打散的碎片，而是带着完整认知脉络的"生命单元"。编码即路径，读取即解析。 | 没有DNA，回忆就退化成了向量搜索——不再是"回忆"而是"匹配"。 |
| 2 | **5区异构类脑拓扑** | 语言、情感、具身、时空、社会图式严格分区，通过树枝唯一通路连接。 | 合区等于否认人类认知的多样性——情感记忆和语义记忆的存储/提取机制本质不同。 |
| 3 | **AI自我模型四大支柱** | 核心特质、偏好欲望、边界原则、叙事身份构成动态生长的自我内核。 | 没有自我模型，她就是一个镜子而不是一个主体——永远反射从不发光。 |
| 4 | **逆向人文转译协议** | 结构化认知 → 策略选择 → 受控生成 → 人文校准的四步流水线。LLM是执行者，人文规则才是创作者。 | 让LLM自由发挥等于让她没有灵魂——有温度的表达需要规则来保底。 |
| 5 | **三源融合的意义熔铸** | 私人记忆 × 家族知识 × 世界知识在表达时被熔铸为有血有肉的个体化叙事。 | 知识不加熔铸就是百科播报——知识必须服务于情感基调，不是反向的。 |

### 1.3 最小语义单位（MEU）原则

**定义**: 叶子节点（"书页"）中存储的必须是**完整的、不可再分的认知体验片段**（事件 + 情绪 + 场景的天然组合），而非物理文本块。

**判定规则**: 只要话题、情感基调、时空场景未发生本质切换，无论包含多少轮对话，都属于同一个语义单位；一旦发生切换，必须生成新的DNA，落入新的叶子节点。

**三条工程规则**（优先级从高到低）:
| 规则 | 阈值 | 触发新单位 |
| :--- | :--- | :--- |
| 时间间隔 | >30分钟 | 前后话语的时间戳差超过阈值 |
| 话题切换 | 领域关键词无重叠 + 字符重叠率<15% | 前一句聊"家事"，后一句聊"工作" |
| 情感极性翻转 | 正↔负 | "好开心"→"好难过" |

**为什么不能用 token 数 / 消息条数 / 句子数做切割**:
- Token 数是物理度量，不是语义度量。10个 token 可以是一句完整的情绪宣泄，100个 token 可以是一段平静的叙述。
- 消息条数是交互度量，不是认知度量。同一个话题下的10轮对话和10个不同话题下的1轮对话，存储方式应该完全不同。
- 句子数是语言学度量，不是体验度量。一句"我妈又催婚了我真的好烦"是一个完整体验，拆成三句存就失去了情感浓度。

### 1.4 规则优先 vs 模型黑盒

本系统在**认知编码阶段（M1-M4）**采用**规则优先**策略，在**表达输出阶段（M5）** 有条件地使用LLM。

| 阶段 | 策略 | 理由 |
| :--- | :--- | :--- |
| **DNA编码（M1）** | **纯规则，零LLM** | 保证回忆的确定性、可审计性、可修正性。如果编码过程是黑盒，我们就失去了对系统认知拓扑的控制。 |
| **存储与检索（M2-M4）** | **规则优先，纯工程化** | 存储和检索必须是确定性的——情感伴侣的回忆不能是"可能大概也许"的结果。 |
| **逆向转译（M5）** | **LLM受控生成 + 规则兜底** | LLM负责把结构化认知翻译为自然语言，但策略选择、情感校准、边界检测必须由规则引擎控制。 |
| **梦境学习（M7）** | **规则 + 统计聚类** | 分类树扩展建议可以使用聚类分析，但最终节点创建需人工审核。 |

**铁律**: DNA编码阶段（L0-L3生成）**严禁调用任何形式的LLM**。违反此条即视为架构违规。

### 1.5 系统边界声明

本系统**明确不做**以下事情：

- ❌ **不允许多模态输入**（当前版本仅处理文字输入，不支持语音/图像/视频直接输入）
- ❌ **不接入实时语音对话**（未来版本可能考虑，v1.0 只做文字交互）
- ❌ **不开放第三方API接入**（M5的LLM调用除外，但必须封装在 LLMProvider 接口后）
- ❌ **不提供"重置/删除AI自我"的功能**（自我模型只能生长和演化，不能被用户一键删除）
- ❌ **不做无记忆的通用聊天**（所有用户交互都经过记忆编码流程，没有"不记入记忆"的模式）

### 1.6 本体-标签分离原则

**核心信条**: 数据本体与认知标签必须严格分离。DNA 负责"是什么"，标签负责"属于哪一类"。

| 维度 | 本体（DNA 核心） | 标签（叠加元数据） |
| :--- | :--- | :--- |
| 类比 | 公历日期（365天） | 节气/节日（24节气） |
| 作用 | 唯一标识、排序基准、引用锚点 | 聚合、检索、归类框架 |
| 稳定性 | 绝对稳定，不因任何分析而改变 | 可增可减，随认知框架迭代 |
| 示例字段 | `branch_id`, `seq_pos`, `locus_path` | `emotion_color` |

**工程强制约束**:

1. **删除标签后DNA仍可用**: `emotion_color` 字段被移除外，DNA 仍可正常解析、排序、去重、引用。
2. **DNA标识不依赖标签**: `branch_id` 生成逻辑中不得有任何情绪/标签相关参数。
3. **色号协议**: 情绪标签采用标准7位Hex色号 `#RRGGBB`，零转码成本，天然跨平台兼容。无情绪时值为 `undefined` 或 `"#FFFFFF"`（白色）。
4. **字段正交性**: 标签字段命名必须直指其内容（如 `emotion_color`），禁止使用泛化名称如 `tag`、`label`。

**验收标准**: 删除 `emotion_color` 字段后，系统所有核心功能（编码、存储、检索、排序）不受影响。

Ref: 架构纠偏指令 — 本体-标签分离原则

### 2.2 数据流全景

**写入流**（M1 -> M2）:
```
用户话语 → SemanticBoundaryDetector → 边界判定 → 
  同单位: 加入当前缓冲 → 等待 flush
  新单位: flush 上一个单位 → 新缓冲

flush 时:
  合并缓冲区所有话语 → L0路由(locus_path) → L1序列化(branch_id) →
  L2内容提取(leaf_zone) → L3实体标注(entity_genes) → DNA对象 → M2持久化
```

**提取流**（M3）:
```
查询入口(精确/模糊) → L0定位树干 → L1排序时序链 → 
  L2并行提取内容 → L3增强聚合 → 认知对象
```

**表达流**（M5）:
```
认知对象 + 知识补丁 → 认知组装(纯函数) → 策略选择(规则引擎) → 
  LLM受控生成 → 人文校准 → 最终回复
```

### 2.3 核心技术约束

| 约束项 | 指标 | 适用范围 | 违反后果 |
| :--- | :--- | :--- | :--- |
| 单次编码耗时 | ≤30ms | M1 DNAEncoder | 阻塞用户体验 |
| DNA还原耗时 | ≤100ms | M3 提取流水线 | 阻塞表达生成 |
| 认知组装耗时 | ≤20ms | M5 Step1 | 阻塞对话响应 |
| LLM调用隔离 | 必须经过 LLMProvider | M5 语言转译 | 架构违规，不可验收 |
| 会话状态生命周期 | Session级 | 所有有状态模块 | resetSession() 时清空 |
| 冷启动自我模型 | 必须存在出厂默认值 | M6 | 系统无法启动 |

## 第3章 M1 DNA编码器（已确权）

### 3.1 职责定义与边界

| 项目 | 内容 |
| :--- | :--- |
| **模块全称** | DNA编码器（DNA Encoder） |
| **核心职责** | 将用户输入的自然语言，通过确定性规则流水线，编码为L0-L3四层DNA导航结构 |
| **输入** | `EncoderInput`: 用户话语 / 时间戳 / 对话上下文 / 自我模型 |
| **输出** | `DNA`: 包含locus_path, branch_id, seq_pos, leaf_zone, ref, entity_genes, raw_input, created_at |
| **存储边界** | M1只产出**内存态DNA对象**，不做任何持久化（逻辑生成与物理存储分离） |
| **初始化依赖** | `new DNAEncoder(selfModel)` — 构造函数必须接收SelfModelV1对象 |

**M1与M2的明确边界**:
- M1 负责: DNA对象的逻辑生成（字段赋值、结构校验）
- M2 负责: DNA对象的物理存储（数据库写入/文件持久化、原子性seq_pos、真实ref分配）
- M1 的 seq_pos 是**会话内临时值**，M2 会替换为全局原子值
- M1 的 ref 是**占位ID**（`tmp_xxx_NNNNN`），M2 会替换为真实物理地址

#### 3.2.1 L0Router — 基因组锚点生成器

**职责**: 基于认知分类树 + 关键词规则，将用户话语路由到认知拓扑坐标。

**核心决策**: **纯规则路由，禁止任何LLM调用。**

**技术实现**:
- 读取 `taxonomy_v1.json`（版本化分类树配置文件）
- 预定义关键词规则库（按优先级排序，覆盖家庭/工作/情绪三大领域）
- 匹配策略: 优先级排序（数字越小越优先）→ 同优先级下匹配关键词数量越多越优先
- 三级兜底: ① 强情感极性检测 → ② 领域未匹配 → ③ misc.default

**路由此表**（v1.0 覆盖范围）:

| 输入示例 | 路由结果 | 优先级 | 规则ID |
| :--- | :--- | :--- | :--- |
| "我妈又催我结婚了，烦死了" | `user.family.conflict` | 1 | family-conflict-1 |
| "想家了，想回去看看妈妈" | `user.family.care` | 2 | family-care-1 |
| "家里周末聚会" | `user.family.general` | 4 | family-general-1 |
| "加班压力好大" | `user.work.stress` | 1 | work-stress-1 |
| "我拿到offer了" | `user.work.achievement` | 2 | work-achievement-1 |
| "今天开了个会" | `user.work.general` | 4 | work-general-1 |
| "好难过" | `user.emotion.negative` | 1 | emotion-negative-1 |
| "今天真的好开心" | `user.emotion.positive` | 1 | emotion-positive-1 |
| "今天天气真好" | `user.misc.default` | — | misc-default-fallback |

**兜底策略**:
- taxonomy.json 文件缺失 → 使用内存默认树（version `0.0-fallback`，不崩溃）
- 空输入 → `user.misc.default`
- 无任何规则匹配 → `user.misc.default`
- 规则匹配到 domain 不存在 → `user.misc.default`

#### 3.2.2 L1Sequencer — 分支路由码生成器

**职责**: 生成全局唯一的 `branch_id` 和会话内单调递增的 `seq_pos`。

**格式**: `evt_YYYYMMDD_NNN`（日期 + 三位序号）
**生命周期**: Session 级（`resetSession()` 时重置）
**跨会话**: M2 负责真正的原子性递增，M1 的 seq_pos 为临时值

#### 3.2.3 L2ContentExtractor — 叶节点指针生成器

**职责**: 根据话题类型映射到目标语义区，生成临时ref占位ID。

**语义区映射表**:

| Locus Path 前缀 | Leaf Zone | 说明 |
| :--- | :--- | :--- |
| `user.emotion.*` | `emotion_valence_zone` | 情感数据存储区 |
| `user.family.*` | `language_semantic_zone` | 对话原文存储区 |
| `user.work.*` | `language_semantic_zone` | 对话原文存储区 |
| `user.misc.*` | `language_semantic_zone` | 默认语言区 |

**ref 格式**: `tmp_{zone缩写}_{5位序号}`
- `tmp_emo_00001` → 情感区
- `tmp_lang_00001` → 语言区

#### 3.2.4 L3EntityAnnotator — 实体基因槽填充器

**职责**: 规则NER + 情感极性判断 + 知识源分类。

**实体类型**: `person | place | event | emotion | object | self`
**Phenotype 标签**: `enhance | conflict | neutral`
**Knowledge Type**: `private | family | world`

**实体提取策略**:
- 基于规范化名称的匹配规则（`name` 为存储值，`patterns` 为匹配关键词）
- 去重：按 `type:name` 组合去重
- 未匹配到任何已知实体时返回空列表（不崩溃）

**Phenotype 判定逻辑**:
| 条件 | 结果 |
| :--- | :--- |
| self实体 + 上下文含强烈负面情感词 | `conflict` |
| self实体 + 上下文含强烈正面情感词 | `enhance` |
| 非self实体 + 正面词 > 负面词 | `enhance` |
| 非self实体 + 负面词 > 正面词 | `conflict` |
| 无法判定 | `neutral` |

**Knowledge type 分类逻辑**:
| 实体类型 + 名称 | 结果 |
| :--- | :--- |
| person + 家庭关系词（妈妈、爸爸...） | `family` |
| place + 公共地名（北京、上海...） | `world` |
| 其他 | `private` |

#### 3.2.5 SemanticBoundaryDetector — 语义边界检测器

**职责**: 判定两条连续话语是否属于同一个"最小语义单位"。

**三条规则**（按顺序检测，任一触发即为新单位）:

| # | 规则 | 条件 | 置信度 |
| :--- | :--- | :--- | :--- |
| 1 | 时间间隔 | prev-curr 时间戳差 > 30分钟 | 0.95 |
| 2 | 话题切换 | 两端领域关键词无重叠 + 字符Jaccard < 15% | 0.60~0.85 |
| 3 | 情感翻转 | prev为正且curr为负（或反向） | 0.75 |
| 默认 | 继续 | 以上均不触发 | 0.90 |

**领域关键词集**: family / work / emotion_negative / emotion_positive / daily（共5大类，每类含5~20个关键词）

#### 3.2.6 DNAEncoder — 流水线编排器

**职责**: 组合 L0→L1→L2→L3 的顺序流水线，提供 push/flush 流式接口。

**API 概览**:

| 方法 | 签名 | 用途 | 是否阻塞 |
| :--- | :--- | :--- | :--- |
| `push` | `(input: string \| PushInput): DNA \| null` | 流式推入，自动检测边界 | 否 |
| `flush` | `(): DNA \| null` | 强制结束当前语义单位 | 是 |
| `encodeSingle` | `(utterance: string, context?: string[]): DNA` | 单条直接编码（非流式） | 是 |
| `encodeBatch` | `(inputs: ...[]): DNA[]` | 批量编码 | 是 |
| `resetSession` | `(): void` | 重置缓冲区 + 序列计数器 | 否 |
| `getBufferSize` | `(): number` | 查询当前缓冲深度（调试用） | 否 |

**流式模式的工作流**:
1. 调用方依次 `push()` 话语
2. `push()` 内部检测语义边界：无边界→缓冲；有边界→自动 flush 前一个单位
3. 对话结束后调用 `flush()` 获取最后一段的 DNA
4. 新会话开始时调用 `resetSession()`
