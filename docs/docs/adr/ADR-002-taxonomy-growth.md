# ADR-002: 认知分类树初始深度与生长策略

- **状态**: 已接受 — 已实施
- **决策日期**: 2026-06-02
- **关联模块**: M1 (L0Router), M7 (L3梦境学习)
- **关联规格书**: SPEC.md §3.2.1

## 背景
认知分类树（taxonomy tree）是 L0 路由的骨架。需要确定其初始复杂度以及如何随着系统的运行迭代演化。

## 选项
- **A**: 一开始设计 4~5 层深度的完整分类树（如 `user.family.mother.cooking_style`）
- **B**: 第一版只做 2 层深（大类 + 子类），后续根据数据积累增量扩展
- **C**: 不做固定树结构，完全由规则动态生成路径

## 决策
**选择 B：初始 2 层深，渐进式生长，版本化管理。**

## 理由
1. **避免过度设计**: 一开始预设大量低频细粒度节点会造成空节点，增加配置工作量却无实际收益.
2. **从数据中生长**: 分类树应该随着对话数据积累而扩展，而不是一开始就焊死一个僵化的骨架.
3. **版本化支持重构**: taxonomy.json 带版本号，每条 DNA 携带 taxonomy_version，未来的记忆重构可以据此识别旧规则下的数据.

## 后果
- **正面**: MVP 快速上线；分类树始终贴合真实数据分布.
- **负面**: 初期大量对话落入 `misc`，覆盖率不够.
- **缓解**: L3 梦境学习可分析 misc 数据，输出分类树扩展建议（人工审核后纳入下一版本）.
- **v1.0 分类树结构**: `user.{family, emotion, work, misc}` 四大类，各含 2~3 个子类.
