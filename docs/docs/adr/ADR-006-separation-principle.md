# ADR-006: 本体-标签分离原则（emotion_color 独立于 DNA 核心标识）

- **状态**: 已接受 — 已实施
- **决策日期**: 2026-06-02
- **关联模块**: M1 (DNA types), M2 (存储), M4/M5 (情感分析)
- **关联规格书**: SPEC.md §1.6, §3.1, §4.2.2

## 背景
在 M1/M2 实现审查中发现，设计文档中对 24 维情绪标签与 DNA 本体的关系描述存在歧义——"24维情绪向量"的措辞暗示了情绪数据是 DNA 的一部分。经项目组深度复盘，需要明确区分"数据本体"与"认知标签"的本质差异。

## 决策

**严格实施本体-标签分离，采用色号作为情绪标签的原生协议。**

### 具体规则
1. **DNA 是日历，情绪是节气**: 情绪标签是叠加在本体之上的认知框架，不参与本体的定义.
2. **色号协议**: 情绪标签采用标准 7 位 Hex 色号 `#RRGGBB` 作为值，零转码成本，天然跨平台兼容。无情绪时值为 `undefined` 或 `"#FFFFFF"`（白色）.
3. **字段命名**: 必须为 `emotion_color`，禁止使用 `tag`、`label` 等泛化名称.
4. **删除标签后 DNA 仍可用**: 移除 `emotion_color` 后，DNA 仍可正常解析、排序、去重、引用.

## 理由
1. **本体稳定性**: DNA 的唯一性约束、版本控制和排序逻辑不能依赖任何可变的认知标签.
2. **零转码成本**: 色号是全球通用的图形学协议，浏览器、设计工具、数据库原生支持，无需维护私有映射字典.
3. **天然支持扩展**: 未来如需表达情绪强度，可直接扩展为 RGBA 格式（如 `#E74C3CCC`），无需新增字段.
4. **所存即所见**: 日志、数据库、API 响应中的色号字符串对人而言自带视觉语义.

## 后果
- **正面**: 架构设计更清晰；情绪维度与 DNA 本体解耦；色号协议消除中间层 Bug 风险.
- **负面**: 需要建立色号 → 情绪标签的映射约定（推迟到 M4/M5 阶段决议）.
- **验证要求**: 必须有自动化测试验证删除 `emotion_color` 后 DNA 核心功能不受影响.
