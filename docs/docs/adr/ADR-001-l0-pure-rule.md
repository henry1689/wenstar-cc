# ADR-001: L0 生成策略——纯规则路由（拒绝LLM方案）

- **状态**: 已接受 — 已实施
- **决策日期**: 2026-06-02
- **关联模块**: M1 (L0Router)
- **关联规格书**: SPEC.md §3.2.1

## 背景
在讨论 M1 L0 基因组锚点（locus_path）的生成策略时，出现了两种方案的分歧。Qwen 最初提议使用"轻量级 LLM + Temperature=0"的方案来生成路径。我方（Claude Code）论证了此方案存在"滑坡风险"——一旦在编码阶段开了"用 LLM 做语义理解"的口子，系统的确定性基石就会动摇。

## 选项
- **A**: 使用轻量级 LLM（Temperature=0）生成 locus_path
- **B**: 使用预定义认知分类树 + 确定性规则路由（关键词模式匹配）
- **C**: 完全规则化 + 实体识别混合方案

## 决策
**选择 B：纯规则路由。** L0 的生成必须是一个完全确定性的、基于规则的函数。给定相同的输入，必须产生完全相同的路径，且逻辑完全透明、可审计。

## 理由
1. **可追溯性**: 每条 locus_path 都可以追溯到触发的规则 ID，而不是 LLM 的隐式推理.
2. **可修正性**: 归类错误时只需调整规则树，无需重新训练或重新推理.
3. **确定性保证**: 相同输入永远得到相同输出，这是"编码即路径"设计哲学的基石.
4. **零 LLM 依赖**: M1 完全不依赖任何外部模型 API，可在离线环境运行，测试成本为零.

## 后果
- **正面**: 系统认知拓扑完全受控；性能极高（微秒级路由）；零 API 成本.
- **负面**: 初始需要人工配置规则树；对未见过的表达方式覆盖率有限（由 misc 兜底处理）.
- **缓解**: 分类树支持版本化渐进式生长；L3 梦境学习可分析 misc 数据驱动规则迭代.
