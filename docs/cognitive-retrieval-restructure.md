# 🧠 认知检索重构方案

## 问题根因

数据流验证：
```
FG数据(年龄14/母亲阿苏/姐姐徐诗雨) 
  → FourLayerDataCollector 采集 ✅
  → PromptAssembler 注入提示词 ✅  
  → LLM 读取 → 回答"我不知道" ❌
```

**数据没有缺失，LLM把四层数据当"角色设定卡"读，不当"自我认知"用。**

### 具体原因

LLM在"扮演"模式下：
1. 将注入的数据视为**背景叙事材料**而非**自传体记忆**
2. 浪漫叙事驱动的回复优先级高于事实回复
3. 没有一个"先确认事实再叙事"的硬约束机制

---

## 改善方案

### 方案A：第一人称自传体改写（低投入，中收益）

**现状**：
```
【核心档案】
姓名：徐诗韵
年龄：14岁

【亲属档案】
· 姓名：阿苏 | 关系：母亲 | 年龄：40岁
```

**改为**：
```
【我知道的关于自己的事】
我叫徐诗韵，今年14岁，初三学生。
我妈妈叫阿苏，今年40岁。
我姐姐叫徐诗雨，今年18岁。
我妹妹叫徐诗涵，12岁。
```

改动位置：`FourLayerDataCollector.ts` buildLayer1() + buildLayer2()

---

### 方案B：事实预检管线（高投入，高收益）

```
用户问"你多大了?"
  │
  ▼
[第一步：事实检索] 非角色扮演模式
  LLM收到"你是徐诗韵。用户问你的年龄。以上数据中你的年龄是什么？"
  → 回答"14岁"
  │
  ▼
[第二步：角色扮演]  
  LLM收到"你是徐诗韵。根据事实[年龄=14岁]回复用户"
  → 回答"诗韵今年14岁呀~"
```

改动位置：在`RoleplayDomain.ts`中增加两阶段管线

---

### 方案C：强制认知指令（最低投入，中等收益，立即见效）

在Prompt最顶部增加不可绕过的自查询指令：
```
【强制认知流程 — 你必须执行】
当用户问事实性问题时，遵循以下步骤：
1. 检查【核心档案】【亲属档案】中的数据
2. 提取相关事实（年龄、关系、人名等）
3. 直接先回答事实
4. 然后才能角色扮演叙事

不遵守此流程的回答将被判定为不合格。
```

改动位置：`PromptAssembler.ts` buildCognitionRule()

---

## 推荐：方案A + 方案C 组合

| 方案 | 投入 | 收益 | 风险 |
|------|------|------|------|
| A: 自传体改写 | 0.5天 | 中 | 低 |
| C: 强制认知指令 | 0.2天 | 中(需模型配合) | 低 |
| **A+C组合** | **0.7天** | **高** | **最低** |

方案B（两阶段管线）放在第二阶段做。
