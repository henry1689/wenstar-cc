# ADR-004: M1-M2 边界分离（逻辑生成 vs 物理存储）

- **状态**: 已接受 — 已实施
- **决策日期**: 2026-06-02
- **关联模块**: M1, M2
- **关联规格书**: SPEC.md §3.1

## 背景
需要明确 M1（DNA 编码器）和 M2（5 大语义区存储适配器）之间的职责边界，防止模块耦合。

## 决策
**采用"逻辑生成与物理存储分离"原则。**

### M1 的职责
- 产出**内存态 DNA 对象**（纯逻辑，无持久化）
- L1: 生成临时 `branch_id` 和会话内 `seq_pos`
- L2: 生成占位 `ref`（`tmp_xxx_NNNNN` 格式）
- 所有中间产物在 `resetSession()` 后丢失

### M2 的职责
- 接收 M1 产出的 DNA 对象，持久化到 5 大区物理存储
- 分配真正的全局原子性 `seq_pos`（通过 counter.json 机制）
- 替换占位 `ref` 为真实物理地址
- 提供 StorageAdapter 接口封装所有存储操作

## 理由
1. **测试独立性**: M1 的测试无需任何数据库或文件系统，纯内存运行.
2. **模块可替换性**: M2 的存储方案（JSON/SQLite/LevelDB）可随时切换，不影响 M1.
3. **职责单一**: 编码器只需关心编码质量，不需要关心数据存在哪、怎么存.

## 后果
- **正面**: 模块边界清晰；M1 测试 47 个全部内存运行，毫秒级完成.
- **负面**: M2 必须自己实现原子性 seq_pos 分配（无法复用 M1 的临时计数器）.
