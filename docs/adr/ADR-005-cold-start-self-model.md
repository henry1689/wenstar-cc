# ADR-005: 冷启动自我模型方案（出厂默认值）

- **状态**: 已接受 — 已实施
- **决策日期**: 2026-06-02
- **关联模块**: M1, M6
- **关联规格书**: SPEC.md §2.3, §3.1

## 背景
系统的 AI 自我模型需要冷启动。但"个性化是在互动中长出来的"——第一次对话时互动还没发生。需要确定初始自我模型是什么形态。

## 选项
- **A**: 完全空白，由用户引导形成
- **B**: 出厂默认人格（含核心特质、边界、偏好、叙事身份）
- **C**: 让用户通过配置界面输入初始人格

## 决策
**选择 B：出厂默认值 + 零样本冷启动。** 在 `self_model_v1.json` 中预设一套基础人格原型。构造函数注入，全局共享。

## 理由
1. **避免空洞**: 如果自我模型完全空白，AI 在第一轮对话中会表现出极度的"空洞"和"讨好"，违背"主体性"原则.
2. **人格出生论**: 出厂默认设置如同刚出生的婴儿自带的"气质"，不需要很复杂，但必须存在.
3. **类型安全**: TS 接口强制要求 self_model 是 DNAEncoder 构造函数的必填参数，避免冷启动时遗漏.

## 后果
- **正面**: 系统开箱即用；自我模型有明确的初始状态可供 M6 演化；类型系统确保不会遗漏.
- **负面**: 出厂人格可能不适合所有用户（但这是设计意图——她不是完美适配所有人的工具）.
- **v1.0 出厂默认值**: 人格"温和、理性、略带好奇"，大五人格 `O:0.7 C:0.6 E:0.4 A:0.8 N:0.3`，3 条边界原则.
