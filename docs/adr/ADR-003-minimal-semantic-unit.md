# ADR-003: 最小语义单位与 push/flush 流式模式

- **状态**: 已接受 — 已实施
- **决策日期**: 2026-06-02
- **关联模块**: M1 (SemanticBoundaryDetector + DNAEncoder)
- **关联规格书**: SPEC.md §1.3, §3.2.5, §3.2.6

## 背景
M1 开发完成后追加的需求：叶子节点的存储粒度必须是"最小语义单位"（MEU），而非物理文本块。原 M1 设计为单次 encode() 调用对应一条 DNA，无法处理多条话语属于同一语义单位的情况。

## 选项
- **A**: 在 M1 中添加语义边界检测器 + 流式缓冲（push/flush）
- **B**: 推迟到 M2 存储层做预处理聚合

## 决策
**选择 A：在 M1 中立即实施。**

## 理由
1. **根源解决**: 如果叶子节点存的是碎片而不是完整体验，DNA 编码得再精确也救不了下游的垮塌.
2. **M1 职责完整性**: DNA 编码器的职责是保证编码质量——如果输入已经是碎片，输出不可能是完整体验.
3. **流式 API 更自然**: push/flush 模式允许调用方自然输入多条话语，编码器自动处理边界检测.

## 后果
- **正面**: 叶子节点存储的是完整体验；API 支持流式输入；自动边界检测无需调用方关心语义切割.
- **负面**: M1 引入有状态缓冲区；回炉成本（重构测试和 Hook）.
- **M1 回炉记录**: [SELF-REPAIR] 新增 SemanticBoundaryDetector + 重构 DNAEncoder 为 push/flush. 所有 83 测试通过.

## 边界检测规则
1. 时间间隔 >30 分钟 → 新单位
2. 话题切换（领域关键词无重叠 + 字符 Jaccard < 15%）→ 新单位
3. 情感极性翻转（正↔负）→ 新单位
4. 默认 → 同一单位
