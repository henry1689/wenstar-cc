# /execute-plan — 执行计划

## 触发条件

已有实现计划需要执行时。

## 前置规则

**每步完成前必须提供验证证据**（参见 `/verify-before-done`）。
"应该没问题"不算完成，要有命令输出。

## 流程

### 第一步：加载计划
读取 `.claude/plans/` 下的计划文件。

### 第二步：按步执行

```
1. 执行变更
2. 运行验证命令 → 粘贴输出作为证据
3. ✅ 通过 → 进入下一步
4. ❌ 失败 → 记录错误、修复、重试、重新验证
```

**证据格式：**
```markdown
### 步骤 N 验证
$ <验证命令>
<输出>
✅ 通过
```

### 第三步：检查点暂停
`[👤 需确认]` 处暂停，向用户展示进度和待确认点。

### 第四步：全面验证
- [ ] `npx tsc --noEmit` 编译通过
- [ ] 全部测试通过
- [ ] 无未提交变更

### 第五步：输出总结

```markdown
## 执行总结

### 完成
- [x] 步骤 1-N

### 问题记录
| 步骤 | 问题 | 解决 |
|------|------|------|

### 下一步
...
```
