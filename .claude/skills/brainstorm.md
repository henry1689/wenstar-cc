# /brainstorm — 设计推演

## 触发条件

用户提出模糊需求、想法或问题时触发。

## 流程

### 第一步：澄清需求
苏格拉底式提问挖出：
1. **目标** — 1 句话
2. **现状** — 现在什么情况
3. **约束** — 时间/技术/资源限制
4. **优先级** — 什么最重要

### 第二步：方案推演
给出 **2-3 个方案**，每个含：
- 名称 / 原理 / 优点(2-3) / 代价(2-3) / 适用场景

### 第三步：推荐
基于优先级推荐并说明原因。

### 第四步：结构化输出

```markdown
## 设计概要

### 目标
...

### 选定方案
...

### 关键决策
| 决策点 | 选择 | 理由 |
|--------|------|------|

### 影响范围
- 涉及文件：
- 涉及模块：
- 风险点：
```
