# /debug — 系统性调试

## 触发条件

遇到 bug 需要根因定位和修复时。

## 流程

### 阶段 1：调查（不跳修复）
收集：复现步骤 / 预期行为 / 实际行为 / 环境 / 错误栈

❌ **禁止**在未找到根因前直接改代码。

### 阶段 2：模式分析
- 对比近期修改（`git diff HEAD~1 --stat`）
- 分析数据流断点
- 二分法缩小范围

### 阶段 3：假设验证
```
假设：问题是因为 X
验证：改 Y 看结果
→ ✅ 成立 / ❌ 不成立
```
持续到找到确切根因。

### 阶段 4：修复与验证
1. 最小改动修复
2. 加回归测试
3. 全部测试通过
4. 输出报告：

```markdown
## 调试报告
### 根因
...
### 修复
文件: `src/foo.ts:42` | 变更: ...
### 验证
- [x] 不复现
- [x] 回归测试
- [x] 全部通过
```
