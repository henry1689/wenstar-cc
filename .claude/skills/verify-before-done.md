## 规则内容

## 核心原则

**"完成了"不是一句声明，是一个有证据的结论。**

你必须在宣布完成之前提供**新鲜的**（刚跑的）命令输出作为证据。

## 验证清单

### 代码变更后

```
✅ 必须做的：
  1. 运行编译检查 → 粘贴输出
  2. 运行相关测试 → 粘贴输出
  3. 如果有运行中的服务，确认没挂

❌ 不可以的：
  "编译应该没问题"            ← 没跑过就是没证据
  "之前测过能跑"              ← 要新鲜的
  "就改了一行注释，不用测"      ← 即使一行也要验证
```

### 具体证据格式

```markdown
### 验证证据

**编译检查：**
```
$ npx tsc --noEmit
✅ 零错误
```

**测试结果：**
```
$ npx vitest run
✅ 12 passed, 0 failed
```

**服务检查：**
```
$ curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/api/status
✅ 200
```
```

## 集成到工作流

这个技能**不单独调用**，而是嵌入到所有技能中：

| 技能 | 验证点 |
|------|--------|
| `/execute-plan` | 每步执行后验证 |
| `/subagent-dev` | 每个子代理完成后验证 |
| `/tdd` | RED→GREEN→REFACTOR 每步验证 |
| `/debug` | 修复后验证不复现 |
| `/review-code` | 审查通过后验证编译+测试 |
| `/finish-branch` | 提交流前全量验证 |

## 例外

只有以下情况可以跳过验证：
- 纯文档/注释修改
- 仅修改非代码文件（.gitignore、LICENSE 等）
- 用户明确说"不用验证"
