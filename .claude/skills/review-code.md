# /review-code — 代码审查

## 触发条件

需要审查代码变更时。

## 流程

### 第一步：准备
```bash
git diff --stat       # 变更概览
git diff              # 变更详情
```

### 第二步：四维审查

**正确性**
- [ ] 逻辑完整？（所有分支）
- [ ] 边界处理？（空/极限/异常）
- [ ] 错误处理恰当？

**设计**
- [ ] 符合模块边界？
- [ ] 接口一致？
- [ ] 不过度设计？

**安全**
- [ ] 输入验证/清理？
- [ ] 注入风险？
- [ ] 敏感数据处理？

**性能**
- [ ] 重复计算？
- [ ] N+1 查询？
- [ ] 循环隐患？

### 第三步：分级

| 级别 | 含义 | 处理 |
|------|------|------|
| 🔴 P0 | 正确性/安全 bug | **必须修** |
| 🟡 P1 | 设计问题 | 建议修 |
| 🔵 P2 | 可读性/风格 | 仅供参考 |

### 第四步：输出

```markdown
## 代码审查
变更: N 文件, +M/-K 行

| # | 级别 | 文件 | 问题 | 建议 |

总体: 🟢 良好 / 🟡 微调 / 🔴 大改
```
