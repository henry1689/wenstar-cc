# /refactor — 安全重构

## 触发条件

代码需要优化结构/消除重复/提升可读性，但不是新增功能。

## 流程

### 第一步：建立安全网
- [ ] 已有测试就运行确认全部通过
- [ ] 没有测试就先写关键测试

### 第二步：识别目标
- 重复逻辑 → 抽取公共函数
- 过长函数 → 拆分
- 过度耦合 → 解耦
- 命名不佳 → 重命名

### 第三步：每次只做一件事
```
1. 选取一个小目标
2. 重构（保持行为不变）
3. 运行测试确认通过
4. 进入下一项
```

### 第四步：输出

```markdown
## 重构报告

### 变更
| 文件 | 重构类型 | 说明 |
|------|---------|------|
| src/a.ts | 抽取函数 | 将重复的 JSON 解析逻辑抽取为 parseData() |

### 验证
- [x] 重构前后测试全部通过
- [x] 行为未改变

### 收益
- 减少重复代码 N 行
- 圈复杂度降低 M 点
```
