# /security-audit — 安全审计

## 触发条件

需要对代码做安全检查时。

## 流程

### 第一步：输入安全
- [ ] 用户输入是否做验证/清理/转义？
- [ ] SQL/NoSQL 注入防护？
- [ ] 路径遍历防护？
- [ ] XSS 防护？

### 第二步：认证与授权
- [ ] 接口是否需要鉴权？
- [ ] 越权访问防护？
- [ ] Token/密钥管理安全？

### 第三步：数据安全
- [ ] 敏感数据存储加密？
- [ ] 日志中是否泄露敏感信息？
- [ ] 传输加密（HTTPS）？

### 第四步：依赖安全
- [ ] 有无已知漏洞依赖？
- [ ] `npm audit` 结果检查

### 第五步：输出

```markdown
## 安全审计报告

| # | 级别 | 问题 | 位置 | 建议 |
|---|------|------|------|------|

总体风险等级: 🟢 低 / 🟡 中 / 🔴 高
```
