# /tdd — 测试驱动开发

## 触发条件

需要实现新功能或修复 bug 且适合 TDD。

## 流程

### RED：先写测试，确认失败
1. 明确行为规范
2. 写测试（正常 + 边界 + 异常路径）
3. 运行 → **确认失败**

### GREEN：最小实现
1. 只写恰好让测试通过的代码
2. 不要过度设计
3. 运行 → **确认通过**

### REFACTOR：测试保护下优化
1. 消除重复、提升可读性
2. 运行 → **确认仍然通过**

### 循环
重复 RED→GREEN→REFACTOR 直到功能完整。

### 验证清单
- [ ] 实现前测试失败
- [ ] 最小实现后通过
- [ ] 重构后仍通过
- [ ] 覆盖正常 + 边界 + 异常
